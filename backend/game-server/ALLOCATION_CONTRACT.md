# Dedicated game-server allocation contract

The allocator must launch one Godot container per match and inject these server-only environment variables:

- `MATCH_ID`: UUID returned in the public assignment.
- `MATCH_SEED`: positive deterministic simulation seed generated by the allocator.
- `SERVER_ID`: Rivet allocation/container identifier returned in the assignment.
- `REGION_ID`: selected Rivet region identifier used for telemetry and match results.
- `BUILD_ID`: `PHASE-05.3-ONLINE-PRODUCTION-COMPLETION`.
- `PROTOCOL_VERSION`: `3`.
- `GAME_PORT`: public ENet UDP port inside the container, default `7000`.
- `CONTROL_PORT`: private HTTP health port, default `7001`.
- `MAX_PLAYERS`: `10`.
- `EXPECTED_PLAYERS`: number of assigned human players; the synchronized start gate waits for this count or the short start deadline.
- `CONTROL_BASE_URL`: private Rivet control-plane URL.
- `GAME_SERVER_AUTH_TOKEN`: unique per-match secret generated by the allocator and shared only with that one container.

The public assignment `match_id`, `server_id`, `host`, `port`, `build_id`, and
`protocol_version` must match these environment values. The Godot server rejects a
hello packet before ticket consumption when the claims differ.

Never place `GAME_SERVER_AUTH_TOKEN` in the Godot Android export, public allocator
response, command line, or deployment report. Provision it with Rivet/CI protected
secrets.
